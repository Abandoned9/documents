    Mybatis一对一查询获取某字段数据异常

    DimitriYoon 10/31/2019 Mybatis持久层Code

    当两张表的主键名称相同时，查询数据异常

    我最近在做一个小项目时，有两张表设计如下：

    # 对应的实体类如下

    package com.dimitri.bv_demo.pojo;

public class BookInfo {

    这里需要注意的是 BookInfo 实体类里的cid属性，在数据库里实际是一个int属性的值，而在实体类里对应的是另一个实体类BookCategory

    # Mybatis配置文件Mapper如下

    <?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" "http://mybatis.org/dtd/mybatis-3-mapper.dtd" >
<mapper namespace="com.dimitri.bv_demo.dao.CategoryDao">
    <resultMap id="bookInfoList" type="com.dimitri.bv_demo.pojo.BookInfo">
        <result property="id" column="id"></result>
        <result property="cover" column="cover"></result>
        <result property="title" column="title"></result>
        <result property="author" column="author"></result>
        <result property="date" column="date"></result>
        <result property="press" column="press"></result>
        <result property="abs" column="abs"></result>
        <association property="cid" javaType="com.dimitri.bv_demo.pojo.BookCategory">
            <id property="id" column="id"></id>
            <result property="name" column="name"></result>
        </association>
    </resultMap>

    <select id="lists" resultMap="bookInfoList">
        SELECT book.id,book.cover,book.title,book.author,book.date,book.press,book.abs,book_category.id,book_category.name
         FROM book,book_category WHERE book.cid=book_category.id ORDER BY book.${id}
    </select>

    <select id="get" resultMap="bookInfoList">
        SELECT book.*,book_category.name FROM book,book_category WHERE book.cid=book_category.id and book.cid=#{id}
    </select>
</mapper>

    这个时候问题就出现了，我在前端获取cid的值，获取到的居然是book表的id值而不是cid值

    跑一遍Test看看log4j里，发现在book表id=1的时候，只查到了1条数据，而我在数据库执行这条语句的时候，明明查询到了4条语句

    查找的过程中发现是两张表的id字段重名，导致mybatis无法正常映射 这时候果断改列明，改完之后，表结构如下

    # 修改完表之后，记得修改一下Mapper文件

    <?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" "http://mybatis.org/dtd/mybatis-3-mapper.dtd" >
<mapper namespace="com.dimitri.bv_demo.dao.CategoryDao">
    <resultMap id="bookInfoList" type="com.dimitri.bv_demo.pojo.BookInfo">
        <result property="id" column="id"></result>
        <result property="cover" column="cover"></result>
        <result property="title" column="title"></result>
        <result property="author" column="author"></result>
        <result property="date" column="date"></result>
        <result property="press" column="press"></result>
        <result property="abs" column="abs"></result>
        <association property="cid" javaType="com.dimitri.bv_demo.pojo.BookCategory">
            <id property="id" column="c_id"></id>
            <result property="name" column="name"></result>
        </association>
    </resultMap>

    <select id="lists" resultMap="bookInfoList">
        SELECT book.id,book.cover,book.title,book.author,book.date,book.press,book.abs,book_category.c_id,book_category.name
         FROM book,book_category WHERE book.cid=book_category.c_id ORDER BY book.${id}
    </select>

    <select id="get" resultMap="bookInfoList">
        SELECT book.*,book_category.name FROM book,book_category WHERE book.cid=book_category.c_id and book.cid=#{id}
    </select>
</mapper>

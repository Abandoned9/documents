    NoSQL和关系型数据库的区别，以及非关系型数据库MongoDB入门和注意事项

    DimitriYoon 12/16/2019 NoSQLMongoDB

    # NoSQL和关系型数据库的区别，以及非关系型数据库MongoDB入门和注意事项

    # NoSQL介绍

    NoSQL(non-relational)指非关系型数据库，最主要的代表有Redis，MongoDB， Oracle BDB 。随着互联网web技术的进步，传统的关系型数据库在处理高并发或者纯动态网站已经有点力不从心。而现代计算机系统产生的庞大数据量，而这些数据很大一部分都是由RDBMS(关系数据管理系统)来处理，这让数据建模和应用编程更加简单。

    # 关系型数据库ACID规则

    在这里谈论到的ACID规则属于事务(transaction)的规则

    # Atomicity 原子性：

    原子性意思是说：transaction里面的操作要么全部做完，要么全部 都不做，其中transaction实际上是一个判断条件，只有在transaction里面所有操作全部成功之后才会执行。

    例如：A银行卡和B银行卡内都有1000元现金，这时A卡向B卡转账500元，这时银行会从A卡扣掉500元，向B卡增加500元，而这两步是同一时间内完成的，只要有一方出现失败，那么转账就不会完成，A卡也不会扣钱，B卡也不会增钱

    # Consistency 一致性：

    一致性的意思是说：所有数据在执行transaction的前后，必须保持原则上的一致。

    例如：A银行卡和B银行卡内都有1000元的现金，那么他们两个账户加起来一共有2000元的现金，A卡向B卡转账指定额数之后，两个账户还是一共只有2000元的现金，不会出现更多。

    # Isolation 独立性：

    独立性是指，当多个用户同时操作数据库时，每个用户会创建一个独立的transaction，它们不会被其他的transaction干扰，即使有两个transaction是并发情况下的，它们也相互隔离，直到其中一个transaction提交完成，才会修改另一个的数据。

    例如：A银行卡向B银行卡转账，那么A卡操作的用户必须确认提交，完成A卡用户的transaction之后，才会修改B银行卡内的现金数额。

    # Durability 持久性：

    持久性相对简单，它是指transaction一旦提交完成，就会永久保存数据，不会因为服务器突然停电或者各种意外情况关闭之后导致数据丢失。

    # NoSQL的用途以及为何使用NoSQL

    NoSQL一般用于超大量数据的存储，而这些类型的数据不需要固定的模式也不需要多余的操作就可以横向扩展。

    现代的各种互联网应用模式会产生太多不需要长期持久保存数据，例如地理位置，用户操作生成的操作日志等等，如果我们想要对这些数据进行查询，使用关系型数据库会因为数据量过多导致各种问题，而NoSQL的发展就能很好的处理这些格式简单却数据量过大的数据。

    # NoSQL数据库分类

    NoSQL数据库发展到现在，也衍生了很多不同的版本，我们可以根据应用的需求使用不同的版本，下面的表格总结出了不同版本之间的不同

    类型 代表 特点
    列存储 Hbase Cassandra Hypertable 顾名思义，是按列存储数据的。最大的特点是方便存储结构化和半结构化数据，方便做数据压缩，对针对某一列或者某几列的查询有非常大的IO优势。
    文档存储 MongoDB CouchDB 文档存储一般用类似json的格式存储，存储的内容是文档型的。这样也就有机会对某些字段建立索引，实现关系数据库的某些功能。
    key-value存储 MemcacheDB Redis 可以通过key快速查询到其value。一般来说，存储不管value的格式，照单全收。（Redis包含了其他功能）
    图存储 Neo4j FlockDB 图形关系的最佳存储。使用传统关系数据库来解决的话性能低下，而且设计使用不方便。
    对象存储 db4o Versant 通过类似面向对象语言的语法操作数据库，通过对象的方式存取数据。
    xml数据库 XML DB 高效的存储XML数据，并支持XML的内部查询语法，比如XQuery,Xpath。

    # MongoDB介绍

    MongoDB是一个基于分布式文件存储的数据库。由C++语言编写。旨在为WEB应用提供可扩展的高性能数据存储解决方案。

    MongoDB 将数据存储以键值对(key:value)的形式存储为一个文件，类似JSON格式，其中value的值可以包含其他文件、数组等等，例如：

    {
	username: "123456",
	password: "123456",
	usergroups: ["123","456"]
}
